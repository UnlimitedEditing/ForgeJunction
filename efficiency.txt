    Priority Order

    1. Viewport culling (biggest immediate win)
    2. Edge path caching
    3. onMouseMove throttling
    4. will-change: transform on the world div
    5. Media lazy-loading

    1. Virtualize node rendering
    Only render nodes visible in the current viewport + a small buffer. Right now every node on
    the canvas renders regardless of whether it's on screen. With 50+ nodes this gets painful
    fast.

    const visibleNodes = nodes.filter(n => {
      const sx = n.position.x * viewport.zoom + viewport.x
      const sy = n.position.y * viewport.zoom + viewport.y
      return sx < vw + 200 && sx + n.size.w * viewport.zoom > -200
          && sy < vh + 200 && sy + n.size.h * viewport.zoom > -200
    })

    2. The edge SVG is the silent killer
    Every edge redraws every frame during pan/zoom. avoidedEdgePath iterates all nodes for every
    edge on every render. With 20 nodes and 15 edges that's 300 node checks per frame. Cache the
    paths and only recompute when node positions actually change (not on every mousemove).

    3. onMouseMove is doing too much
    Right now it's running port proximity detection across every node on every mouse movement.
    Move this to requestAnimationFrame throttling or a debounced loop.

    4. Stop re-rendering the whole canvas on viewport change
    The viewport state change triggers a full re-render of the world div and all its children. Use
     useMemo for the node list and edge SVG paths. The transform should be CSS-only — use
    will-change: transform on the world div so the GPU handles panning without layout reflows.

    5. Image/video nodes
    Unload media from nodes outside the viewport. Use loading="lazy" on images. For video nodes,
    <video> elements outside the viewport should have their src removed and restored on
    visibility.
