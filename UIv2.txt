Card selection behavior:

Clicking a card calls setSelectedRender(id) (toggles off if already selected)
Selected card gets a distinct highlight: ring-2 ring-brand or similar
The full RenderCard detail (progress bar, media, log) only expands for the selected card. Unselected cards show a collapsed summary: thumbnail (if resultUrl exists), workflow slug, status badge, and elapsed time. This keeps the feed scannable.

In-progress card selected:
When the selected card's status is 'queued', 'active', or 'streaming':

Look up the workflow from useWorkflowStore().workflows using render.workflowSlug to find its output type (check supports_txt2img, supports_txt2vid, supports_txt2wav etc.)
Call useSourceMediaStore().setPendingSource(...) with a synthetic pending entry that communicates intent — use mediaType inferred from the workflow type: video → 'video', audio → 'audio', else 'image'
This will cause WorkflowSelector's existing source-media filter to activate automatically, narrowing the list to workflows that accept that media type
Show a small notice on the card: "⏳ Queued as source — workflow list filtered"

When the selected in-progress render completes:

Automatically call setFromRender(resultUrl, mediaType) so the result is immediately available as source input

Remove: The ← Prev / Next → carousel navigation, the selectedId local state, all tabbing logic. Replace with the feed as described.
Do NOT change: RenderCard internals, MediaPreview, UseAsSourceButton, CancelButton, the queue store's enqueue/process logic.
