STEP 4 — Simplified Floating Prompt Input Pane
File: src/components/PromptEditor.tsx
Redesign PromptEditor from a full-panel layout into a compact floating bar styled similarly to Sora's input — a small prompt field with an expandable "advanced" drawer above it.
New structure (from bottom up):
┌──────────────────────────────────────────────────┐
│  [Advanced drawer — hidden by default]            │
│  Parameters · Negative · Media dropzones          │
│  Raw prompt preview (collapsible within)          │
├──────────────────────────────────────────────────┤
│  [Main bar]                                       │
│  [📎][⚙] [prompt textarea         ] [→ Submit]  │
└──────────────────────────────────────────────────┘
Main bar (always visible):

Compact textarea (1–3 lines, auto-grows up to ~4 lines, then scrolls)
⚙ button: toggles the advanced drawer open/closed
📎 or + button: quick source media attach (opens a small floating picker or directly triggers the dropzone)
→ Submit button (or ↵): same as current submit logic — no changes to handleSubmit
Keyboard shortcut: Enter submits, Shift+Enter newlines (unchanged)
Paste-detection for Telegram prompt format (unchanged)

Advanced drawer (slides up from above the main bar when ⚙ is clicked):

Contains, in order: ParameterBar, MediaDropZone (when slots exist), negative prompt field, raw prompt view toggle
Styled as a floating card with backdrop blur and rounded-t-xl top edge
Animation: slide up/down with a CSS transition (max-height transition or transform: translateY)
The drawer should have a max-height and be internally scrollable if content overflows

Styling notes:

The entire component should have bg-neutral-900/90 backdrop-blur-md with a top border border-t border-white/10
Fits the absolute bottom-0 left-0 right-0 positioning that STEP 3 established
Keep existing Tailwind utility classes and theme tokens (brand, arc, muted, etc.)
The WorkflowHeader (showing selected workflow + Change button) moves INTO the main bar as a small workflow-name chip to the left of the textarea

What to preserve unchanged:

All the logic: handleSubmit, handlePaste, buildRawPrompt, Telegram parsing, validation warnings, missing slot detection
ParameterBar, MediaDropZone, RawPromptView sub-components — just relocated into the drawer
All store interactions (usePromptStore, useWorkflowStore, useRenderQueueStore, useSourceMediaStore)

Remove: The scrollable full-panel layout wrapper. The component no longer needs flex flex-col h-full since it's a floating overlay now.
