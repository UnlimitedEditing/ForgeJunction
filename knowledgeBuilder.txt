Knowledge Base API
Overview
The Knowledge Base API allows you to create knowledge repositories that can be searched and used for AI-powered chat interactions. Each knowledge base is organized into domains that can contain facts, documents, and crawled web content.

Understanding Domains
A domain is simply the name of a knowledge base instance—it is not related to internet domain names in any way. Think of it as a namespace or identifier for your knowledge collection.

For example, you might create domains named product-docs, faq-database, or company-policies.

Domain Ownership
The API token that creates a domain becomes its owner. Domain ownership determines who can perform write operations:

Read operations (search, chat) — available to all users
Write operations (create, crawl, add fact) — require domain ownership
When you create a domain using the Create Domain endpoint, your API token is automatically recorded as the owner. Only requests using that same token can modify the domain's contents.

Create Domain
Create a new knowledge base domain. You will become the owner of this domain and can then add facts, crawl URLs, and manage its contents.

URL
POST https://caPxkya125x.graydient.ai/api/v3/knowledge
POST Parameters
metadata
(optional)	
Optional metadata object to attach to the domain (e.g., description). The API will automatically add owner_user_id and created_at.

name
(required)	
The name for your new knowledge base domain. This will be used in all subsequent API calls to reference this domain. Use lowercase letters, numbers, and hyphens only.

CURL example
#!/bin/bash
curl -X POST \
  -H "Authorization: Bearer __YOUR_TOKEN__" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-knowledge-base",
    "metadata": {
      "description": "Product documentation knowledge base"
    }
  }' \
  https://caPxkya125x.graydient.ai/api/v3/knowledge
Response format
{
  "data": {
    "name": "my-knowledge-base",
    "metadata": {
      "description": "Product documentation knowledge base",
      "owner_user_id": 123,
      "created_at": "2026-01-15T10:30:00Z"
    },
    "status": "domain created"
  }
}
Search Knowledge Base
Search for relevant information within a knowledge domain.

URL
POST https://caPxkya125x.graydient.ai/api/v3/knowledge/:domain/search
POST Parameters
n_results
(optional)	
Number of results to return. Default is 10, maximum is 20.

query
(required)	
The search query to find relevant information in the knowledge base.

CURL example
#!/bin/bash
curl -X POST \
  -H "Authorization: Bearer __YOUR_TOKEN__" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "artificial intelligence",
    "n_results": 5
  }' \
  https://caPxkya125x.graydient.ai/api/v3/knowledge/mydomain/search
Response format
Returns an array of search results with similarity scores:

{
  "data": {
    "domain": "my-knowledge-base",
    "query": "artificial intelligence",
    "results": [
      {
        "id": "fact-artificial-intelligence-refers-to...",
        "document": "Artificial Intelligence (AI) refers to computer systems...",
        "distance": 0.42,
        "metadata": {
          "type": "fact",
          "source": "api",
          "user_id": 123,
          "timestamp": "2026-01-15T10:30:00Z"
        }
      }
    ]
  }
}
Chat with Knowledge Base
Have a conversation with an AI that has access to domain-specific knowledge.

URL
POST https://caPxkya125x.graydient.ai/api/v3/knowledge/:domain/chat
POST Parameters
query
(required)	
Your question or prompt for the knowledge base chatbot. The chatbot will use the domain's knowledge to answer.

CURL example
#!/bin/bash
curl -X POST \
  -H "Authorization: Bearer __YOUR_TOKEN__" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What are the main applications of AI in healthcare?"
  }' \
  https://caPxkya125x.graydient.ai/api/v3/knowledge/mydomain/chat
Response format
{
  "data": {
    "domain": "my-knowledge-base",
    "query": "What are the main features?",
    "response": "Based on the knowledge base, the main features include real-time sync, offline mode, and collaborative editing.",
    "sources": [
      {
        "id": "fact-main-features",
        "document": "The product supports real-time synchronization...",
        "distance": 0.35
      }
    ]
  }
}
Crawl URL (Owner Only)
Crawl a website and add its content to your knowledge base domain. Requires domain ownership.

URL
POST https://caPxkya125x.graydient.ai/api/v3/knowledge/:domain/crawl
POST Parameters
max_depth
(optional)	
Maximum crawl depth from the starting URL. Default is 2.

restrict_prefix
(optional)	
URL prefix to restrict crawling. Only pages starting with this prefix will be crawled. Defaults to the provided URL.

url
(required)	
The URL to crawl and add to the knowledge base.

CURL example
#!/bin/bash
curl -X POST \
  -H "Authorization: Bearer __YOUR_TOKEN__" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/ai-resources",
    "restrict_prefix": "https://example.com/ai-resources",
    "max_depth": 3
  }' \
  https://caPxkya125x.graydient.ai/api/v3/knowledge/mydomain/crawl
Response format
{
  "data": {
    "domain": "my-knowledge-base",
    "url": "https://example.com/docs",
    "restrict_prefix": "https://example.com/docs",
    "max_depth": 2,
    "status": "crawl started"
  }
}
Add Fact (Owner Only)
Add a fact to your knowledge base domain. Requires domain ownership.

URL
POST https://caPxkya125x.graydient.ai/api/v3/knowledge/:domain/fact
POST Parameters
metadata
(optional)	
Optional metadata object to attach to the fact. The API will automatically add source, user_id, and timestamp.

text
(required)	
The fact text to add to the knowledge base.

CURL example
#!/bin/bash
curl -X POST \
  -H "Authorization: Bearer __YOUR_TOKEN__" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "GPT-4 is a large language model developed by OpenAI",
    "metadata": {
      "category": "AI Models",
      "confidence": "high"
    }
  }' \
  https://caPxkya125x.graydient.ai/api/v3/knowledge/mydomain/fact
Response format
{
  "data": {
    "domain": "my-knowledge-base",
    "text": "GPT-4 is a large language model developed by OpenAI",
    "metadata": {
      "category": "AI Models",
      "confidence": "high",
      "source": "api",
      "user_id": 123,
      "timestamp": "2026-01-15T10:30:00Z"
    },
    "status": "fact added"
  }
}
